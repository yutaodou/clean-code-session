Clean Code

课程的回顾，本次的内容 程序的两个消费者，计算机和人，机器对代码的clean程度没要求，只有作为代码的作者和维护者的人才提出了clean code要求 机器可以使用无法理解的命名，深层嵌套

"Any fool can write code that a computer can understand. Good programmers write code that humans can understand."

所以，当我们要给出clean code定义的时候，我们一般会有两个角度： 可读性，也即是容易理解 修改的难以程度（可维护性、可扩展性），也就是添加新功能是的难易程度

"Always code as if the guy who ends up maintaining your code will be a violent psychopath who knows where you live."

- Martin Golding

同team的人在对这一段垃圾代码咆哮的时候， 完了又git blame时， 写代码不知关乎项目，还可能关乎人身安全 如何写clean code? 一些大家都了解的，比如magic number, duplication, small method, small class 这里列了几个high level的guideline
